## Linear issue workflow (required after every delegated task)

When finishing work for a Linear issue (e.g. `IIK-6`):

1. Implement the change and open a PR linked to the issue (branch name should include the issue identifier).
2. Post a comment on the **Linear issue** (not only on GitHub) that:
   - @mentions the issue assignee (`Muhammat Lio Pratama`)
   - Lists each acceptance criterion with `[x]` or `[ ]` based on what you completed

   - States blockers or manual steps left for the human
3. Do **not** mark the issue Done yourself — leave status for human review after merge.
4. If you cannot edit the issue description, put the updated checklist in the Linear comment.
